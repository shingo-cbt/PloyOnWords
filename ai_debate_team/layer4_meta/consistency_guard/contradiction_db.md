# Contradiction DB — 矛盾パターンデータベース

## 目的
よくある矛盾パターンをあらかじめ定義し、ConsistencyGuardが素早く検出できるようにする。

---

## 矛盾パターン一覧

### PATTERN-01: 定義逆転型
**発生条件**: 同じ言葉を異なるターンで逆の意味で使う
```
例:
ターンN: 「〇〇とは△△である」（ポジティブな定義）
ターンM: 「〇〇が問題なのは□□だから」（同じ言葉を別文脈で使用）
```
**検出キーワード**: 同じ名詞が複数ターンで異なる文脈に登場
**対処**: F-01（定義のすり替え）を使っている場合は意図的なので、
変化が「連続的」かどうかを確認する。急な逆転はREDフラグ。

---

### PATTERN-02: 数字矛盾型
**発生条件**: 同じ対象について異なる数値・割合を提示する
```
例:
ターン2: 「70%の人が〇〇を経験している」
ターン5: 「ほとんどの人は〇〇を気にしていない」
（70%とほとんどが矛盾する可能性）
```
**検出キーワード**: %, 割合, 多数, ほとんど, 少数, まれ
**対処**: 数字の対象・条件が異なるなら矛盾ではない。
「〇〇について70%」と「△△についてほとんど」は別物。

---

### PATTERN-03: 因果逆転後の矛盾型
**発生条件**: F-05（因果の逆転）を使った後、元の因果関係に戻る
```
例:
ターン3: 「実は〇〇が△△の原因ではなく、逆です」（因果逆転）
ターン5: 「〇〇があることで△△が起きる」（元に戻っている）
```
**検出キーワード**: 「実は逆」「原因は」「引き起こす」「なぜなら」
**対処**: 即YELLOW以上。どちらの因果を採用するか決定して修正。

---

### PATTERN-04: 権威の自己否定型
**発生条件**: 権威を使った後、その権威を疑う発言をする
```
例:
ターン2: 「〇〇大学の研究では〜」（権威の借用）
ターン4: 「専門家の意見だけが正しいわけじゃないですけど」
（同じ「専門家」を否定している）
```
**対処**: YELLOW。ターン4の発言を「権威への言及なし」に修正。

---

### PATTERN-05: スケール矛盾型
**発生条件**: スケールアップした後、個人レベルの話に戻る
```
例:
ターン4: 「これは社会全体の設計の問題です」（スケールアップ）
ターン5: 「だからあなた個人が〇〇すべきでは」（スケールダウン）
```
**対処**: スケールは一方向に動かす。社会レベルに行ったら
社会レベルで締めるか、「だから個人レベルでも〜」と橋渡しを入れる。

---

### PATTERN-06: 同情と批判の混在型
**発生条件**: F-14（同情の悪用）で被害者性を強調した後、批判的な発言をする
```
例:
ターン2: 「〇〇だって、選んでそうなったわけじゃない」（同情）
ターン4: 「でも〇〇には問題がある面もある」（批判）
```
**対処**: 擁護コンテンツでは批判的発言は原則NG。
もし入れる場合は「でも」を使わず、「それとこれは別の話として」とフレームを変える。

---

## クリーン状態の定義

以下をすべて満たしていればGREEN：
1. 定義が変化している場合、変化が段階的（急変しない）
2. 数字の対象・条件が各ターンで一貫している
3. 因果の方向性が途中で逆転していない（または逆転を明示している）
4. 権威を使った場合、その権威を後で否定していない
5. スケールの変化に橋渡し発言がある
6. 感情系の詭弁（同情・恐怖）の方向性が一貫している
