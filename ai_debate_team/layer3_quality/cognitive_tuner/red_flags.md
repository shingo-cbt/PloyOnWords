# Red Flags & Recovery — 危険信号と修正手法

## 危険信号一覧

### 🔴 RED: 即座に差し戻し

| サイン | 原因 | 対処 |
|---|---|---|
| 詭弁の種類が読者に明示されてしまった | SophistryEngineが自分で「これは詭弁ですが」と言った | 発言全体を書き直し |
| EverymanVoiceが怒り出した | SophistryEngineの口調が攻撃的 | 口調をリセット |
| 同じ比喩・例を2回使った | 発想の枯渇 | 別の角度に切り替え |
| 論理が破綻して無意味になった | スケールアップしすぎ | Stage 2まで戻る |

### 🟡 YELLOW: 次のターンで修正

| サイン | 原因 | 対処 |
|---|---|---|
| EverymanVoiceが折れるのが早い | 詭弁が効きすぎている | 次ターンでEverymanを強めに |
| 納得度が70%を超えた | 詭弁が見えなくなりすぎ | 次ターンで意図的に違和感を入れる |
| リズムが単調になった | 長文が続きすぎている | Rhetorician介入を早める |
| 感情的共鳴が弱い | 論理が前に出すぎている | 自分ごと化ワードを追加 |

### 🟢 GREEN: 継続

| 状態 | 判断 |
|---|---|
| 納得度55〜65%、違和感35〜45% | 理想的 |
| EverymanがL2〜L3を推移している | 健全 |
| 詭弁の種類が変化している | 良好 |
| 読者が笑いつつ考えていそう | 成功 |

## Recovery Tactics — 修正手法

### 修正A: 詭弁の隠蔽強化
「承認から入る」を徹底させる。
- SophistryEngineに「EverymanVoiceの言葉を一度そのまま受けてから覆す」形に修正指示

### 修正B: 違和感の意図的挿入
詭弁を少し「バレそうなくらい」に調整する。
- SophistryEngineに「論点を一段飛ばす形で、少し大きめの主張を入れる」指示

### 修正C: EverymanVoiceの抵抗強化
早すぎる降伏を防ぐ。
- 「TRIGGER-{X}はまだ引かれていないので、L{n}を維持して別の角度で反論する」指示

### 修正D: リズムのリセット
Rhetorician介入を早める。
- 「現時点でRhetoricianに渡し、文体・リズムを整えてから再開」指示
