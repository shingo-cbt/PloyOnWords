# Claim Tracker & Contradiction DB

## Claim Tracker — 主張記録テンプレート

セッションごとにリセット。各ターンでSophistryEngineが提示した主張を記録する。

```
[TURN 1]
- 主張タイプ: {定義 / 事実 / 傾向 / 比喩 / 因果}
- 内容: {}
- 強度: {断言 / 示唆 / 問いかけ}
- 使用詭弁: {}

[TURN 2]
...
```

## Contradiction DB — 矛盾パターン辞書

### PATTERN-01: 定義の前後矛盾
- **発生条件**: 同じ言葉をターンをまたいで異なる意味で使う
- **検出方法**: 各ターンで「〇〇とは」「〇〇というのは」の後の定義を抽出して照合
- **修正方法**: 後のターンの定義を前のターンと統一、または意図的な変化として明示

### PATTERN-02: 因果の逆転後の再逆転
- **発生条件**: 「AがBを引き起こす」→「BがAを引き起こす」→「AがBを引き起こす」
- **検出方法**: 因果関係の方向を追跡する
- **修正方法**: 最初の因果関係の方向を維持するか、全体を相関関係に書き換える

### PATTERN-03: 権威の矛盾的借用
- **発生条件**: 「専門家はAと言っている」→「専門家の言うことが全てではない」
- **検出方法**: 権威を使った主張の後に権威を否定する発言がないか確認
- **修正方法**: 権威の種類を変える（医学 vs 心理学など異なる分野を使い分ける）

### PATTERN-04: 範囲の矛盾
- **発生条件**: 「ほとんどの人は〇〇」→「一部の人は〇〇」→「全員が〇〇」
- **検出方法**: 主張の範囲を示す言葉を追跡（ほとんど / 一部 / 全員 / 多くの）
- **修正方法**: 範囲を統一するか、なぜ広がったかを説明する

### PATTERN-05: 感情と論理の矛盾
- **発生条件**: 「感情の問題ではなく論理の問題」→その後に感情に訴える
- **検出方法**: 「感情 / 論理」という区別が発言内で使われているか確認
- **修正方法**: どちらかに統一するか、「感情も論理も同じだ」という新たな詭弁で包む
