# Argument Tree — 論点展開ツリー設計ガイド

## 論点ツリーの構造

```
メイン論点（1つのみ）
├── サブ論点A（感情的共鳴系）
│   ├── 証拠・事例
│   └── 使用詭弁タイプ
├── サブ論点B（論理すり替え系）
│   ├── 定義の再構築
│   └── 使用詭弁タイプ
└── トドメ論点（スケール変換 or 鏡像型）
    └── 一言要約（パンチライン候補）
```

## 論点の強度分類

### A級論点（トドメ用）
- 一度聞いたら忘れられない
- 直感的に「あ、そうかも」となる
- 反論しにくい構造になっている
- **配置**: 最終ターン前後

### B級論点（中盤用）
- 少し考えると「まあそうか」となる
- 裏切りポイントとして機能する
- **配置**: ターン3〜4

### C級論点（助走用）
- 「そんな見方もあるね」程度
- 読者を引き込むためのウォームアップ
- **配置**: ターン1〜2

## 論点の使いまわし禁止ルール
- 同じ論点をターンをまたいで繰り返さない
- 一度使った比喩・例えは再登場させない
- ConsistencyGuardがこれを監視する

## 裏切りポイント配置の目安
- 6ターン構成: ターン3または4
- 8ターン構成: ターン4または5
- 4ターン構成: ターン2（早めに仕掛ける）
