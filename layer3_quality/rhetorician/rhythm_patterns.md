# Rhythm Patterns — 文体リズムのパターン集

## なぜリズムが重要か
読者は「内容を読む」前に「リズムを感じる」。
リズムが良いと、詭弁への警戒心が下がる。
リズムが悪いと、どんなに良い詭弁も「説教」に聞こえる。

---

## パターン1: ジャブ型（SophistryEngine 序盤向け）
```
短文。短文。少し展開する文。でも結論は短く。
```
**例**:
「そうですよね。わかります。でも、一個だけ聞いていいですか。
それって、誰が決めたんでしたっけ。」

**効果**: テンポが良く、最後の短文が引っかかりを作る

---

## パターン2: 包み込み型（感情的共鳴向け）
```
長めの前置き（共感）。でも。短い核心。
```
**例**:
「気持ちはすごくわかるんですよ、自分も最初はそう思ってたし。
でも、よく考えると、それって本当にそうですかね。」

**効果**: 警戒心を下げてから本題に入る

---

## パターン3: 畳み掛け型（終盤向け）
```
短文。短文。短文。少し長い文で締める。
```
**例**:
「だって、みんなやってるじゃないですか。気づいてないだけで。
そう考えると、批判する側の方が、実は気づいていないことが多い気がしませんか。」

**効果**: 速度感で相手を飲み込む

---

## パターン4: 沈黙型（揺らぎの演出向け）
```
問いかけ。（改行）（改行）短い相手の反応。
```
**例**:
「だとしたら、あなたが批判してるのは、実は自分自身ってことになりませんか。」

「...」

「それ、言い返せないんですよね、だいたいの人が。」

**効果**: 間が読者に「自分もそうかも」と考えさせる

---

## パターン5: EverymanVoice 揺らぎ型
```
言いかけ。止まる。認める。でもまだ抵抗。
```
**例**:
「いや、それは...うーん。まぁ、その部分はそうかもしれないけど。
全部は認めたくないんだよな。」

**効果**: 読者が「自分と同じ気持ちだ」と同一化する

---

## リズムチェックリスト
- [ ] 長文が3文以上続いていないか
- [ ] 「です・ます」が連続しすぎていないか
- [ ] 「...」や改行が有効に使われているか
- [ ] 核心的な詭弁の文は短くなっているか
- [ ] 笑える瞬間が1ターンに1回あるか
